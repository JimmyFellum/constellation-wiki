# Benchmark

Automated pipeline to measure the quality and stability of the Constellation Wiki agentic cycle.

---

## Purpose

The benchmark verifies how many cycles of the inner loop (HB → Builder → Reviewer → Evaluator) are needed to produce a stable wiki, and measures the drift between automated runs and a hand-built baseline. Recurring drift across runs indicates architectural defects in the void scaffold.

---

## Structure

```
benchmark/
├── corpora/                        # Raw source sets used as test input
│   └── harness-engineering/        # 7 documents on agent harness engineering
├── pipeline/                       # Automation scripts
│   ├── orchestrator.py             # Runs the full pipeline against a sandbox
│   ├── agent_runner.py             # Launches a single role via Claude Code CLI
│   ├── metrics.py                  # Extracts metrics from a completed wiki
│   └── report.py                   # Cross-run comparison table
├── runs/                           # Metrics extracted from each run (committed)
│   └── run-001-manual/
│       └── metrics.json
└── sandboxes/                      # Live wiki instances generated by the pipeline (gitignored)
    └── run-001-manual/             # Wiki built manually — source of the baseline
```

---

## How a run works

1. **Setup** — a fresh copy of the void scaffold is created in `sandboxes/run-NNN/` and the corpus sources are copied into `sources/raw/`
2. **Heartbeat** — discovers and registers all sources, creates canonical manifests
3. **Builder** — reads manifests and raw sources, generates knowledge pages
4. **Reviewer** — QC on Builder output, applies mechanical fixes, emits correction requests
5. **Evaluator** — tests answerability of the wiki, emits semantic improvement requests
6. Steps 3–5 repeat until no open requests remain or `--max-cycles` is reached
7. **Metrics** — extracted from the sandbox filesystem and saved to `runs/run-NNN/metrics.json`

---

## Running the pipeline

```bash
cd benchmark/pipeline

# Full automated run
python orchestrator.py --corpus harness-engineering --run-id run-002 --model claude-sonnet-4-6 --max-cycles 3

# Single role (manual step-by-step)
python agent_runner.py heartbeat <sandbox_path> --model claude-sonnet-4-6
python agent_runner.py builder   <sandbox_path> --model claude-sonnet-4-6
python agent_runner.py reviewer  <sandbox_path> --model claude-sonnet-4-6
python agent_runner.py evaluator <sandbox_path> --model claude-sonnet-4-6

# Comparison report
python report.py
```

## Metrics

Each `runs/run-NNN/metrics.json` contains four metric groups:

| Group | Key metrics |
|---|---|
| `pages` | page_count, page_count_by_type, mean_page_length, multi_source_pages_pct |
| `graph` | relation_density, generated_from_coverage, bidirectional_rate, orphan_pages |
| `sources` | raw_sources_total, manifests_created, source_coverage_pct, pages_per_source |
| `cycle` | build_invocations, requests_emitted_total, open_requests_at_end |

---

## Baseline — run-001-manual

The reference baseline was produced by a manual run on the harness-engineering corpus (2 full cycles: HB + Build + Review + Eval × 2).

| Metric | Value |
|---|---|
| pages | 27 |
| entities / concepts / playbooks / synthesis | 8 / 14 / 4 / 1 |
| relation_density | 5.19 |
| generated_from_coverage | 1.0 |
| bidirectional_rate | 1.0 |
| orphan_pages | 3 |
| source_coverage_pct | 100% |
| open_requests_at_end | 3 |
| cycles | 2 (HB + Build + Review + Eval × 2) |
